# ModeratorBOT 🤖

Універсальний, швидкий та незалежний модуль модерації для Telegram-груп.
Розроблено спеціально для екосистеми "ОБЩЕПІТ".

## Що вміє бот?
- 🚫 **Spam Detector**: Блокує заборонені слова, комерційні посилання, ботів, капс та надлишок емодзі.
- 🌊 **Flood Detector**: Рахує частоту повідомлень і видає `warning`, `mute` або `ban`.
- 🔄 **Fake Distribution Detector**: Знаходить дублікати повідомлень та реферальні лінки.
- ⚙️ **Rule Engine**: Всі санкції (видалення, мут, бан, кік, попередження) гнучко налаштовуються у JSON-правилах.

## Вимоги
- Python 3.10+
- SQLite3
- Docker & Docker Compose (опційно для ізольованого запуску)

## Швидкий старт (Локально)
1. Створіть віртуальне середовище і встановіть залежності:
   ```bash
   ./scripts/setup.sh
   # (Або: python3 -m venv venv && source venv/bin/activate && pip install -r requirements.txt)
   ```
2. Відредагуйте `.env` та вкажіть ваш `TELEGRAM_BOT_TOKEN`.
3. Запустіть бота:
   ```bash
   python -m src.main
   ```

## Швидкий старт (Docker)
1. Заповніть `.env` файл токеном.
2. Запустіть контейнер:
   ```bash
   docker-compose up -d --build
   ```
3. Логи можна дивитись через: `docker-compose logs -f`

## Команди Адміністратора
Усі команди можуть використовуватись ТІЛЬКИ адміністраторами групи. Бот повинен мати права адміністратора (видалення повідомлень, блокування користувачів).

- `/status` — Перевірити чи бот активний
- `/ban` (Reply) — Заблокувати користувача
- `/mute <seconds>` (Reply) — Заборонити писати на N секунд
- `/rules` — Перевірити активність движка правил

## Структура конфігів (`config/`)
- `config.yaml` — Налаштування таймерів, лімітів, та "білих списків" чатів/посилань.
- `rules.json` — Основна бізнес-логіка, яка поєднує детектори з діями (Actions).
- `blacklist.json` — Список слів для спам-детектора. На льоту завантажується системою.

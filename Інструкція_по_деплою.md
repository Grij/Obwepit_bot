# Інструкція з деплою бота "ОБЩЕПІТ" на VPS (Ubuntu/Debian)

Ця інструкція передбачає, що у вас є доступ до VPS сервера по SSH (наприклад, під користувачем `root`).

## Крок 1: Завантаження коду на сервер

Вам потрібно перенести папку `src` на ваш сервер. Найпростіше це зробити через SFTP (наприклад за допомогою програми FileZilla або Cyberduck).

1. Підключіться до сервера по SFTP.
2. Створіть папку `/opt/obshepit_bot` на сервері.
3. Скопіюйте всі файли з вашої локальної папки `Projects/ОБЩЕПІТ/src` у папку `/opt/obshepit_bot` на сервері.
4. Перейменуйте файл `.env.example` на `.env` і вставте туди справжній токен від вашого бота:
   ```env
   BOT_TOKEN=1234567890:AAH_XXXXXXX...
   ```

## Крок 2: Налаштування середовища (через SSH)

Підключіться до сервера через термінал (SSH):

```bash
ssh root@IP_ВАШОГО_СЕРВЕРА
```

Виконайте наступні команди:

```bash
# Переходимо в папку з ботом
cd /opt/obshepit_bot

# Встановлюємо python3-venv, якщо його ще немає
sudo apt update
sudo apt install python3-venv -y

# Створюємо віртуальне середовище
python3 -m venv venv

# Встановлюємо залежності бота
./venv/bin/pip install -r requirements.txt
```

## Крок 3: Налаштування запуску як служби (systemd)

Щоб бот працював завжди, навіть після перезавантаження сервера, ми зробимо його системною службою.

1. Скопіюйте файл `obshepit_bot.service` у папку системних служб:
```bash
cp /opt/obshepit_bot/obshepit_bot.service /etc/systemd/system/
```

2. Оновіть список служб, увімкніть автозапуск бота і запустіть його:
```bash
systemctl daemon-reload
systemctl enable obshepit_bot
systemctl start obshepit_bot
```

3. Перевірте, чи бот успішно запустився (має бути статус `active (running)`):
```bash
systemctl status obshepit_bot
```

## Готово!
Бот працює! Тепер:
1. Додайте бота в адміністратори вашого чату "Общепіт".
2. Обов'язково дайте йому права "Додавати користувачів" (Add Users) / "Схвалювати заявки" (Approve Join Requests).
3. Протестуйте подачу заявки з іншого акаунту.

## Актуалізація для Docker-продакшну (2026-03-08)

Поточний прод розгортається через Docker Compose на VPS:

- Шлях: `/opt/gemini/services/obwepit_bot`
- Команда деплою web: `docker compose up -d --build web`

### Обов'язкові ENV для web-доступу (Google OAuth)

У файлі `.env` мають бути:

```env
BOT_TOKEN=...
SECRET_KEY=...
GOOGLE_CLIENT_ID=...apps.googleusercontent.com
GOOGLE_CLIENT_SECRET=...
ALLOWED_EMAIL=8704421@gmail.com
POST_SIGNATURE_LINK=https://bio.site/obwepit
```

### Smoke-check після деплою

1. `docker compose ps` — контейнер `obwepit_web` у статусі `Up`.
2. `GET /login` — сторінка входу відкривається.
3. `GET /auth/google` — повертає `302` (редірект на Google з `client_id`).
4. На головній сторінці видно графік `Динаміка Апрувів`.
5. У верхньому меню видно вкладку `Модератор`.

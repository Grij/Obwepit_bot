# Технічний план реалізації Telegram-бота "ОБЩЕПІТ"

Цей документ описує технічну реалізацію бота для автоматизації прийому заявок у чати "Общепіт".

## 1. Стек технологій
- **Мова програмування:** Python 3.10+
- **Бібліотека:** `aiogram` 3.x (найсучасніша асинхронна бібліотека для Telegram API)
- **Зберігання даних:** Словник у пам'яті (In-memory dict) для тимчасового збереження капчі (хто яку відповідь має надати). Для поточного сценарію це найшвидший та найоптимальніший варіант.
- **Хостинг:** Власний VPS сервер (Linux). Скрипт працюватиме як фонова служба через `systemd`.

## 2. Структура коду бота
Весь код буде розташовано в папці `Projects/ОБЩЕПІТ/src/`:
- `main.py` — запуск бота.
- `config.py` — налаштування та читання токену з файлу `.env`.
- `handlers.py` — логіка обробки заявок (`chat_join_request`) та відповідей користувачів.
- `captcha.py` — логіка генерації прикладів ("два додати сім" -> 9).
- `storage.py` — керування тимчасовими станами користувачів (збереження їхніх правильних відповідей та `chat_id`).
- `requirements.txt` — бібліотеки для встановлення.

## 3. Логіка роботи (User Flow)
1. Користувач тисне "Подати заявку" у вашому Telegram-каналі чи групі.
2. Бот перехоплює апдейт `chat_join_request`.
3. Бот генерує капчу (наприклад: 5 + 7 = 12 -> "п'ять додати сім"), запам'ятовує, що від юзера X очікується відповідь "12" для чату Y.
4. Бот надсилає приватне повідомлення юзеру X з проханням розв'язати приклад. *(Примітка: Telegram дозволяє ботам писати юзерам, які подали заявку на вступ до каналу/групи, де бот є адміністратором).*
5. Користувач цілеспрямовано пише боту цифру "12".
6. Бот отримує повідомлення від користувача, перевіряє зі збереженим числом 12.
7. Якщо все вірно: бот викликає метод `approve_chat_join_request`, пише юзеру напутнє слово "Відповідь правильна, доступ відкрито!".
8. Якщо помилка: бот викликає метод `decline_chat_join_request`, повідомляє про помилку "Неправильно. Заявку відхилено.". Значення очищується з пам'яті.

## 4. План деплою на VPS
1. Підключитися до сервера по SSH.
2. Створити папку для бота: `mkdir -p /opt/obshepit_bot`
3. Закинути туди всі файли з папки `src/`.
4. Створити віртуальне середовище Python: `python3 -m venv venv`
5. Встановити залежності: `./venv/bin/pip install -r requirements.txt`
6. Створити файл `obshepit_bot.service` у `/etc/systemd/system/`.
7. Запустити службу: `systemctl enable --now obshepit_bot`.
